O problema com o jeito antigo

Montar material no PowerPoint custa caro em tempo e em qualidade. Quatro dores se repetem em toda área do banco:

01 — tempo

Você passa mais tempo alinhando caixas e ajustando cores do que pensando no conteúdo. A forma rouba o tempo do raciocínio.

02 — viewport

O que cabia no notebook vaza no projetor: texto cortado, rodapé sumido, imagem espremida. Cada tela é uma surpresa.

03 — Office

O arquivo só abre direito na mesma versão do programa; as fontes trocam de máquina; no celular vira um borrão.

04 — marca

Cada pessoa interpreta a identidade Itaú do seu jeito. O laranja varia, a tipografia varia, o banco aparece diferente.

a tese

Qualquer ituber, em 1 arquivo

Qualquer pessoa do banco, de qualquer área, cria material profissional em minutos — sem designer, sem instalar nada, sem PowerPoint. O resultado é um único HTML que abre em qualquer navegador e vira link, e-mail ou PDF. Este bloco é o único spotlight do documento: passe o cursor.

PowerPoint × Slideless

PowerPoint

  • Arquivo pesado, preso a uma versão do programa
  • Layout engessado em molde de slide
  • Sem acessibilidade real
  • Some ou quebra no celular
  • Marca aplicada na mão, sempre diferente

Slideless

  • HTML universal — abre em qualquer navegador
  • Direção de arte feita para o conteúdo
  • Acessibilidade WCAG AA embutida
  • Responsivo: lê igual no celular e no projetor
  • Identidade Itaú e modo escuro nativos
  • Compartilha por link ou exporta em PDF

um formato certo para cada entrega

Seis formatos

Todos com a mesma régua de qualidade. Cada um carrega a inspiração de uma referência de design que o mundo já reconhece. Passe o cursor: os cards inclinam em 3D (W23).

ao vivo · apresentar

Deck

Slides com navegação por teclado, fragmentos que revelam por clique e tela cheia. Para all-hands, comitês e pitches projetados. Inspiração: keynotes da Apple.

consultar · documentar

Handbook

Manual com sidebar, índice navegável e scrollspy. Para procedimentos e bases de conhecimento que se voltam a consultar. Inspiração: o handbook da GitLab.

reunir · catalogar

Hub

Portal de cards categorizados que abrem detalhes na própria página. Para reunir links, times, produtos ou recursos. Inspiração: o Apple Developer.

contar · narrar

Scrollytelling

História que se revela ao rolar, com gráficos que reagem à narrativa. Para contar uma jornada ou um resultado com tensão e ritmo. Inspiração: o NYT Upshot.

navegar · microsite

Site

Microsite de várias abas com navegação por hash, sem recarregar a página. É o formato deste documento. Inspiração: Linear e Vercel.

imprimir · relatório

Report

Editorial denso com sumário executivo, seções numeradas e notas de rodapé, otimizado para virar PDF impecável. Inspiração: McKinsey e Goldman Sachs.

os 3 invariantes

A fiação que nunca sai

Todo bloco já vem com: (1) reduced-motion sempre; (2) @supports + estado-final-base, para um Chrome travado nunca sumir nada; (3) só animar transform/opacity. Single-file, sempre.

Um drop-in, por dentro

O W14 (odômetro) é um exemplo de como cada bloco já chega blindado. O valor final fica em texto real no HTML — fallback sem JS — e a tira de dígitos é clipada a height: 1em para a coluna 0–9 nunca vazar:

css · W14
/* a tira de dígitos PRECISA de altura + clip */
.od-digit { display:inline-block; width:1ch; height:1em; overflow:clip; }
.od-reel  { display:flex; flex-direction:column; transition: transform 900ms; }
.od-reel > span { height:1em; }
@media (prefers-reduced-motion: reduce) { .od-reel { transition:none; } }
i

Por que verbatim?

Improviso de ponta é frágil: vira tímido ou quebra na intranet. O LLM copia daqui e troca só o payload — o número real, a série do gráfico, a manchete.

Validados em harness

Cada componente é creditado pelo validador determinístico através de uma signature (W14-odometer, W17-sticky-stack, …). O catálogo inteiro roda com zero erros e zero warnings no harness. Esta é a única palavra com glitch no documento — parcimônia é regra.

o que toda entrega respeita, sem exceção

Garantias inegociáveis

Single-file, degradação graciosa e gates automáticos. As regras não são sugestões: são verificadas por dois scripts a cada geração.

Arquivo único, de verdade

CSS e JavaScript embutidos no HTML. CDN apenas para Google Fonts e Chart.js — nada de servidor, nada de pasta de assets. Um só arquivo para guardar, versionar e mandar por link.

✓

Boot script antes do CSS

O tema é aplicado antes da primeira pintura — modo escuro sem flash, mesmo em conexão lenta. Padrão verificado pelo gate B2.

Degradação graciosa por camadas

Todo gesto de ponta tem rede. O estado final é o CSS base; a animação só acontece dentro de um @supports. Num navegador antigo, ou com Chart.js bloqueado pela intranet, o conteúdo aparece inteiro — só sem o brilho.

sem @supports

O estado-final-base entrega o conteúdo legível. Nada some.

sem JS / CDN

Números em texto real no HTML; o canvas vira aviso "indisponível offline".

reduced-motion

Todo movimento desliga; o documento fica estático e perfeitamente usável.

Dois gates, a cada geração

Antes de entregar, a skill roda dois scripts e itera até ambos passarem limpos:

shell
# 1. validador determinístico (estrutura, parti, anti-patterns)
python scripts/validar.py outputs/slideless-site.html   → 0 erro(s)
# 2. teste de render headless (overflow, odômetro, duplicação, scroll-x)
python scripts/smoke.py outputs/slideless-site.html     → SMOKE PASS

O validador vê a estrutura; o smoke vê o que a estrutura não vê — quebra de render num Chromium headless. Juntos, pegam desde um <script> aninhado que mata o JS em silêncio até a coluna 0–9 de um odômetro vazando.

As cinco garantias

  1. Arquivo único — CSS e JS embutidos; um só HTML para guardar e compartilhar.
  2. Modo escuro nativo — claro e escuro de fábrica, sem flash ao abrir.
  3. WCAG AA — foco visível, navegação por teclado, contraste e leitor de tela.
  4. Conteúdo real — a skill nunca inventa dados internos do Itaú; sem fonte, não gera número.
  5. Documento web, não slide — nada vaza da viewport e nada fica preso a um molde de PowerPoint.

não é template — é direção de arte

O parti

Antes de qualquer HTML, a skill declara um parti: a decisão de design daquele documento. Sete eixos + um campo de renúncia. É o que impede dois documentos Itaú de compartilharem fingerprint.

Os sete eixos de decisão

registro

O tom: de institucional-impresso sóbrio a revista-interna calorosa.

kit

A tipografia curada; fontes com cara de IA são banidas pelo validador.

capa

O gesto de abertura — broadsheet, full-bleed, split assimétrico.

superfície

A materialité: flat, grain, glow localizado, aurora. Não quita ambição.

motion

Estático, editorial ou cinemático — colado como bloco aditivo.

ambição

A1 contido, A2 elevado, A3 extraordinário. Este site é A3.

assinatura

O momento que define o documento. Aqui: o morph entre views (W5).

O músculo simétrico

O campo nao-vai-ter é verificado: se o documento declara que não terá hover-lift, o validador falha caso o padrão apareça. Ambição declarada também é cobrada — A3 sem momento-wow substantivo é erro.
